커리어 목표

핵심: “잘 돌아가는 환경”을 만드는 개발·품질 통합형 개발자(Dev×QA) (기능 개발 + 테스트 자동화 + 배포 품질 관리)

  • 단기(0~3M): 온보딩 · 도메인/API 파악 · 문서↔실응답 불일치 정리 · 공통 오류응답 초안 · 작은 PR 연속
  • 중기(3~12M): 서버 API 자동화 테스트 구축 · DB 컨테이너 기반 통합테스트 · CI 연동 · JWT/권한/WebSocket 회귀 시나리오 · 문서-코드 동기화
  • 장기(1~3Y): 품질지표(에러율·MTTR) 운영 · 배포/릴리즈 표준화 · 보안 기본 가이드 · 개발+QA 공동 책임

일하는 방식: 구조부터 점검 → 타임박싱/우선순위 → 작은 PR → 재발 방지 문서화

용어 한 줄 설명 보기
  • 개발·품질 통합(Dev×QA): 기능 개발 + 자동화 테스트 + 품질 지표/릴리즈 관리
  • Postman/Swagger: API 테스트 도구 / API 문서
  • 자동화 테스트: JUnit·MockMvc·RestAssured로 서버 API 검증
  • Testcontainers: 테스트 시 실제 DB/브로커를 컨테이너로 기동
  • CI: GitHub Actions로 푸시마다 빌드·테스트 자동 실행
  • MTTR: 평균 복구 시간
